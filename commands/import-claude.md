description: "One-shot ingest of a Claude.ai or ChatGPT data export (zip) into the vault — alias-classified conversation captures, factual LIT-* mirrors, and linked concept stubs."
---

Read the `import-claude` skill at `skills/import-claude/SKILL.md`, then run the workflow. Pre-requisite: the export zip has been staged via `bash scripts/import-claude.sh` into `<vault>/.import-staging/<timestamp>-claude/`. Flags: `--scan` (inventory only, no writes), `--dry-run` (full classification pass in memory, default), `--apply` (execute writes after explicit `yes`), `--since YYYY-MM-DD`, `--project <name>`, `--resume` (reads `Claude-Memory/import-claude-state.json`). The skill alias-classifies every conversation against `Claude-Memory/aliases.yaml`, writes full-fidelity captures to `01-Conversations/<project-mirror>/YYYY-MM-DD-<slug>.md` (`owner: human`), factual mirrors to `02-Sources/LIT-conversation-<slug>-<date>.md`, and cautious concept stubs to `03-Concepts/`. Never overwrites `owner: human` files, never writes `05-Projects/*/<project>.md` index files. Commit prefix `[bot:import-claude]`. For daily per-session capture use `/save`; for Claude Code JSONL sessions use `/backfill`.
