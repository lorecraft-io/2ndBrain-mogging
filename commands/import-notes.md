Read the `import-notes` skill at `skills/import-notes/SKILL.md`, then run the workflow. Pre-requisite: `bash scripts/import-notes.sh` has verified `pandoc` + `xlsx2csv` are installed. Flags: `--source <path>` (required — the folder holding exported notes), `--kind <apple|onenote|notion|evernote|raw>` (optional hint; auto-detected otherwise), `--scan` (inventory only), `--dry-run` (classification pass in memory, default), `--apply` (execute after explicit `yes`), `--project <name>` (force-tether every file to one project), `--resume` (reads `Claude-Memory/import-notes-state.json`). The skill converts non-markdown files to markdown, validates each, alias-classifies against `Claude-Memory/aliases.yaml`, and routes per decision table: external reads → `02-Sources/SRC-<date>-<slug>.md`; atomic ideas → `03-Concepts/<slug>.md`; project-tied material → `05-Projects/<project>/<subfolder>/<slug>.md` (NEVER the project index file); tasks → refused, user is pointed at `06-Tasks/` + the Obsidian Tasks plugin UUID format. Commit prefix `[bot:import-notes]`. For single-note capture use `/save`; for Claude.ai/ChatGPT exports use `/import-claude`.
